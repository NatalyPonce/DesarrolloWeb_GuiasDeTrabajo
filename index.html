Descripción de la materia

Desarrollo web I es una materia de segundo año de ingeniería de software. Es impartida por el catedrático Guillermo Calderón. Se divide en HTML y CSS durante 5 semanas y en las restantes se abordará JS. Hasta el momento, se han estudiado conceptos de maquetación web, diseños responsivos, teoría de cajas y otros temas que son de gran utilidad en el diseño web.

Guías de trabajo

Guía 1

En esta guía de trabajo, se abordarón los conceptos básicos de HTML y CSS. Se evaluó que el documento HTML contará con la estructura correcta. A su vez, se demostraba el uso de estilos con hojas externas como embebidos. Por último, se debía hacer uso del conocimiento sobre selectores y propiedades CSS para darle un diseño adecuado a las páginas web.

  • Página de una serie
  • Ejemplo 2
  • Estilos embebidos
  • Estilos con css
  • Página de paises

Guía 2

Se estudiaron y aplicaron los selectores avanzados. También, se usaron pseudo clses y pseudo elementos de CSS. Por último, el alumo debía crear su propia página web haciendo uso del modelo de caja y todas sus propiedades.

  • Biografía del alumno
  • Página de película
  • Tablero de ajedrez

Guía 3

En esta guía de trabajo, se utilizaron los conceptos de posicionamiento en CSS (absoluto y estático) y se estudiaron sus diferencias. A su vez, se aplicaron los conceptos de visualización de elementos. Por último, los alumnos aprendieron maquetación de diseños fijos y fluidos.

  • Posicionamiento de cartas 1
  • Posicionamiento de cartas 2
  • Galería de imagenes
  • Index de imagenes
  • Menú horizontal
  • Oscares
  • Planes

Guía 4

Se exploró el uso de formularios y el uso de la etiqueta viewport. A su vez, se usaron medias queries para aplicar estilos a diferentes tipos de anchos de pantalla. Por último, se aplicaron los conceptos de creación de páginas responsivas.

  • Página de Login
  • Creación de perfil
  • Página de confirmación de envío
  • Galeria de imagenes
  • Index de planes
  • Ejemplo de Login 2
  • Uso de mediaqueries
  • Planes

Guía 5

En esta actividad práctica se puso a prueba el dominio de los elementos básicos de JavaScript. A su vez, se buscaba evaluar que el alumno supiera integrar y generar código HTML con JS. Por último, se utilizaron las sentencias de flujo y de interacción con el usuario.

  • Ejemplo1: interacción básica con el usuario.
  • Ejemplo2: creando tablas con JavaScript.
  • Ejemplo3: número aleatorio.
  • Ejemplo4: operaciones básicas.

FODA

Fortalezas

  • He mejorado la capacidad de ordenar y estructurar correctamente código HTML y CSS.
  • Aprendi a dominar el uso de media queries para crear diseños responsivos y fluidos.
  • Aprendí el uso correcto de propiedades de caja y como usarlos para estilizar una página web.

Debilidades

  • El diseño de páginas web atractivas y de fácil uso para los usuarios.
  • El aprendizaje de conceptos importantes como grid y el uso de flexbox para estructuras más complejas.
  • Compatibilidad entre diferentes navegadores (safari, Firefox, Internet explorer).

Ventajas

  • Diseño de páginas web flexibles y fluidas.
  • Uso de herramientas colaborativas como Github serán muy utiles para el futuro y para nuevos proyectos
  • Resolución de problemas y capacidad de investigar en caso de necesitar fuentes externas.

Amenazas

  • Falta de tiempo no me permitía profundizar en CSS y HTML.
  • Poco conocimiento de Github fue un gran desafío para entregar mis primeras guías y para trabajos grupales.
  • En mi computadora ciertos estilos se visualizaban diferente o no funcionaban como debían.

Retroalimentación sobre el curso

Lo que me gustó: en lo personal, disfrute mucho la parte de Flexbox y teoría de caja. Me motivó a seguir investigando para crear sitios web con estructuras cada vez más complejas. A su vez, me gustó mucho que en las clases se hacían ejemplos prácticos que ayudaban a comprender la teoría.

Lo no me gustó: las clases durante el fin de semana resultaban pesadas. Además, era desafiante que la jornada fuera tan larga, incluso con el descanso de por medio.

Retos asumidos: honestamente, casi no enfrente ninguna dificultad. Solo tuve problemas de compatibilidad con mi computadora, pero lo pude arreglar con una extensión. Por último, se me dificultó el uso de Github, pero vi tutoriales y eso me ayudó a comprender.

Sugerencias para mejorar: distribuir más las jornadas. Aunque sean virtuales, el contenido del curso no se ve afectado, pues siempre veriamos los ejemplos de las clases y el desarrollo.
